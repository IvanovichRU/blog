Código Fuente

Una vez abierto nuestro archivo en VS Code, podemos comenzar con el código. Para poder realizar operaciones con el sistema de archivos de nuestro sistema operativo (temas que se mencionaron en clase), debemos utilizar una librería estándar incluida con Node.js llamada fs (abreviatura para file system o sistema de archivos). Esta librería nos dará acceso a funcionalidades como leer archivos, modificarlos, borrarlos, crearlos, etc. En Node.js existen múltiples maneras de manejar librerías, pero iremos con la más común, la de CommonJS, si gustas, puedes investigarlo más a fondo, aquí lo veremos superficialmente, la primer línea de código es la siguiente: 

const fs = require("fs");

En la anterior línea de código están sucediendo dos cosas a grandes razgos:

  • Creamos una variable constante mediante el uso de la palabra clave const llamada fs. esta variable contendrá toda la funcionalidad de la librería del mismo nombre, accederemos a ella mediante el uso de la notación de punto, por ejemplo: fs.funcionAleatoria().
  • Podemos acceder a estas funcionalidades de la librería gracias a la segunda acción que cumple la línea de código, require("fs"). La función require, la cual estamos llamando en este extracto de código, permite asignar todas las funcionalidades y propiedades de una librería mediante el nombre que le pasemos como argumento, en este caso, "fs".

Una herramienta muy útil de VS Code al usar JavaScript es su autocompletado, si estás trabajando con VS Code, en la siguiente línea de código, experimenta escribiendo fs. y mira las sugerencias y/o opciones de autocompletado que VS Code nos da, proporciona el nombre de la propiedad o función además de dar información útil y una breve explicación del uso de la opción o sugerencia, puedes navegar las opciones con las flechas arriba y abajo de tu teclado.

Autocompletado de VS Code

Para nuestro programa utilizaremos la función writeFileSync(). Si escribimos esta función y abrimos sus paréntesis para llamarla, el autocompletado de VS Code nos dirá los argumentos que espera esta función, es decir, los que deberemos pasarle dentro de los paréntesis para que la función opere correctamente.

Detalles de la función readFileSync

Podemos ver que en primer lugar aparece file: fs.PathOrFileDescriptor, esta notación es de otro lenguaje basado en JavaScript llamado TypeScript, básicamente nos indica de qué tipo es el dato que debemos pasarle en este espacio a la función, podríamos investigar exactamente qué es fs.PathOrFileDescriptor pero puedo asegurarte que incluye el tipo string que es una cadena en JavaScript. Para denotar una cadena podemos utilizar las comillas dobles " ", las comillas simples ' ' o el "backtick" ` `: 

const cadenaConComillasDobles = "cadena1";
const cadenaConComillasSimples = 'cadena2';
const cadenaConBackticks = `cadena3`;

Recomiendo utilizar las comillas dobles o simples y dejar el backtick para más adelante en casos específicos donde conviene su uso. Entonces, sabemos que debemos pasarle una cadena, esta cadena será el nombre del archivo que crearemos, por ahora podemos nombrarlo "archivo.txt". La extensión txt en verdad no es necesario pero es convención nombrar archivos de texto plano con esta extensión. Llevamos lo siguiente: 

const fs = require("fs");
fs.writeFileSync("archivo.txt", )

Tras escribir la coma luego de nuestra cadena, VS Code nos dirá el siguiente argumento:

El siguiente argumento

El siguiente argumento es data: string | NodeJS.ArrayBufferView, aquí explícitamente podemos ver que acepta string, por lo que se entiende que debemos pasarle una cadena más, si lees los detalles del autocompletado se explica que este argumento son los datos que se escribirán al archivo en su momento de creación, aquí es donde podemos pasarle el mensaje a escribir al momento de su creación. Escribe el mensaje que desees yo escribiré "¡Hola, mundo!". 

const fs = require("fs");
fs.writeFileSync("archivo.txt", "¡Hola, mundo!")

Si somos curiosos y escribimos una coma más luego del segundo argumento, veremos que se nos indica un argumento más, options?: fs.WriteFileOptions | undefined hay una pequeña diferencia de este argumento a los dos anteriores, ¿La notas...? Justo antes de los dos puntos hay un signo de interrogación ?, este signo de interrogación indica que este argumento es opcional y podemos decidir no pasárselo a la función, esta operará sin problemas. 🎉¡Y listo!🎉, nuestro código está terminado, aunque no lo creas, es todo lo necesario para escribir un archivo con un mensaje dentro.

Ejecución del Código

Ahora, ¿Cómo lo ejecutamos?, es sencillo, si está usando VS Code podemos abrir una consola dentro del programa desde el menú del programa en la sección de "Terminal", damos click en la opción "Nueva Terminal" o hacemos la combinación de teclas Ctrl+Shift+ñ y se abrirá una consola en la parte inferior de VS Code, esta consola ya debería encontrarse en la carpeta de trabajo, puesto que es la carpeta que tenemos abierta en el editor.

Una vez abierta la consola basta con escribir node practica1.js en la consola y dar Enter para ejecutar el programa, sustituyendo practica1.js con el nombre que le hayas puesto al archivo con tu código fuente. Tras dar Enter podremos ver que se creó nuestro archivo en la barra del lado izquierdo (el explorador de archivos de VS Code). Si le damos click se abrirá y podremos ver el contenido que le pasamos a la función fs.writeFileSync()
